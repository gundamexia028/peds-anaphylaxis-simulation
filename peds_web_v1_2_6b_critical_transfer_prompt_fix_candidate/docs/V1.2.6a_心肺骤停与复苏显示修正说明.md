# V1.2.6a 心肺骤停与复苏显示修正说明

本版本基于 V1.2.6，不改变25分标准评分路径，仅修正心肺骤停与复苏相关的条件性危重分支逻辑。

## 1. 25分主路径不变

标准25分仍包括：停用可疑药物并保留静脉通路、呼救、ABC评估、开放气道并给氧、体位管理、连接监护、测血压、肌注肾上腺素、快速扩容、第一次复评、雾化支扩、糖皮质激素剂量输入、第二次复评、告知家属、SBAR交接。

CPR、球囊面罩加压给氧、联系高级生命支持仍为条件性危重分支记录，不纳入25分主分。

## 2. 心肺骤停前后显示逻辑

V1.2.6中可出现“CPR已记录但HR仍显示220/min”的显示矛盾。V1.2.6a修正为：

- 濒危休克/心肺骤停前：可显示HR 180–220/min、SpO₂极低、BP极低、RR极快。
- 一旦进入心肺骤停状态：前台生命体征显示被覆盖为：
  - SpO₂：无可靠波形/不可测；
  - HR：无脉搏/不可测；
  - RR：无有效自主呼吸；
  - BP：不可测。

## 3. CPR适用条件

CPR仅在系统判断达到心肺骤停或近似心肺骤停状态时作为正确危重分支处理。气道梗阻、低氧或低血压本身不等于CPR。

## 4. 复苏成功/ROSC逻辑

心肺骤停前，肌注肾上腺素仍是严重过敏反应的一线关键处置。

心肺骤停后，不再要求“心肺骤停后再次肌注肾上腺素”作为复苏成功条件。系统将普通病房护士应完成的基础复苏关键操作设定为：

1. CPR；
2. 球囊面罩加压给氧/人工通气；
3. 联系高级生命支持。

三者完成后，系统记录基础复苏启动正确，并通过叙事方式进入高级生命支持接手后的ROSC状态：恢复可触及脉搏，但仍需持续通气支持、监护和转运。

## 5. 新增或修正字段

- cardiac_arrest
- cardiac_arrest_time_sec
- cpr_required
- resuscitation_required
- resuscitation_in_progress
- resuscitation_initiated_correctly
- resuscitation_rosc
- resuscitation_rosc_time_sec
- post_arrest_care_required
- final_vitals_display

## 6. 研究分析原则

主分析仍使用25分主路径得分。心肺骤停、CPR、BVM、ALS和ROSC作为条件性危重分支指标，用于探索性分析和过程性安全缺陷描述。
