# V1.2.6g 标准流程延迟计分与普通考核终止条件修正版

本版本基于 V1.2.6f，保留 25 分标准主路径、糖皮质激素剂量输入、双复评、气道危重分支、心肺骤停未 CPR 死亡事件、高级支持当前指征逻辑，仅调整普通考核评分与终止规则。

## 1. 评分逻辑

标准路径仍为 25 分。评分原则调整为：

- 按标准流程顺序完成：获得该项目满分；
- 延迟完成：获得该项目一半分，向下取整；
- 未完成：该项目 0 分，不额外扣主分；
- 错误操作、用药安全事件、危重分支仍单独记录为安全指标或结局指标，不再从主分中额外扣减。

延迟完成的定义：某一标准操作在其后续标准操作已经被尝试后才补做，即判定为延迟操作。

## 2. 普通考核终止条件

普通考核不再要求所有标准动作均完成后才结束。满足以下条件即结束普通考核：

1. 有效第一次复评；
2. 有效第二次复评；
3. 已完成家属沟通；
4. 已完成 SBAR 交接。

普通考核结束后，尚未完成的标准项目按 0 分处理，不允许继续补做获取分数。

## 3. 危重分支不受普通终止条件限制

如进入以下情况，仍按危重分支或失败分支终止：

- 心肺骤停后未立即 CPR：记录死亡事件；
- CPR + 球囊加压给氧 + 联系高级生命支持：危重抢救后转入 PICU；
- 严重肾上腺素过量：失败结局；
- 超过最大时间仍未完成：超时结局。

## 4. 报告字段

新增/保留字段包括：

- `standard_assessment_completed`
- `ordinary_exam_terminal`
- `standard_pathway_full_score`
- `delayed_actions`
- `penalties_not_subtracted_from_main_score`

主分 `score` 不再扣除 `penalties`，`penalties` 仅作为安全记录字段保留。
