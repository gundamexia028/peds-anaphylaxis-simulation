# V1.2.1 研究质控与分析字段说明

## 1. 版本目标

V1.2.1 是“研究质控与分析版”，用于将虚拟仿真训练数据转化为可用于多院区横断面分析、培训前后比较、论文统计和护理部汇报的数据结构。

本版重点包括：

- 阶段完成状态识别；
- 阶段顺序提醒；
- 重复作答识别；
- 数据有效性标记；
- 管理员后台质控概览；
- 研究分析指标自动导出。

## 2. 数据质量字段

| 字段 | 说明 | 推荐用途 |
|---|---|---|
| `record_validity` | 有效 / 需核查 | 主分析纳入前筛查 |
| `validity_note` | 具体异常说明 | 数据清洗日志 |
| `is_repeated_attempt` | 同一参与者同一阶段是否重复 | 重复记录剔除或敏感性分析 |
| `attempt_order` | 同一阶段第几次记录 | 保留首次记录或比较重复趋势 |
| `stage_sequence_status` | 阶段顺序是否合理 | 判断基线、培训、后测是否可配对 |
| `previous_completed_stages` | 当前记录之前已完成阶段 | 流程追踪 |
| `missing_required_previous_stage` | 缺少的前序阶段 | 现场组织质控 |

## 3. 研究分析字段

### 识别能力

| 字段 | 说明 |
|---|---|
| `recognition_time_sec` | 首次完成停药/呼救/ABC评估中任一识别相关操作的时间 |
| `recognized_anaphylaxis` | 是否出现识别相关操作 |
| `stop_infusion_done` | 是否停止输液/停药 |
| `call_help_done` | 是否呼救 |

### 关键处置能力

| 字段 | 说明 |
|---|---|
| `epinephrine_given` | 是否有效执行肌注肾上腺素 |
| `epinephrine_time_sec` | 肾上腺素给药时间 |
| `epinephrine_dose_correct` | 肾上腺素剂量是否正确 |
| `oxygen_given` | 是否给氧 |
| `monitoring_connected` | 是否连接监护 |
| `bp_assessment_done` | 是否测血压/灌注评估 |
| `fluid_resuscitation_given` | 是否静脉补液 |

### 用药安全

| 字段 | 说明 |
|---|---|
| `epinephrine_under_dose` | 是否低剂量，判定为无效操作 |
| `epinephrine_over_dose` | 是否超过0.3 mg |
| `serious_medication_error` | 是否出现严重用药/安全事件 |

### 流程完整性

| 字段 | 说明 |
|---|---|
| `abc_assessment_done` | 是否完成ABC评估 |
| `reassessment_done` | 是否完成复评 |
| `reassessment_count_analysis` | 复评次数 |
| `family_communication_done` | 是否家属沟通 |
| `sbar_handoff_done` | 是否SBAR交接 |
| `critical_step_completion_count` | 关键步骤完成数 |
| `critical_step_completion_rate` | 关键步骤完成率 |

## 4. 数据库兼容性

本版本不新增 Supabase 顶层表字段，避免 `training_records` 表因缺少字段而写入失败。新增字段保存在：

1. `full_report.research_indicators`；
2. `full_report.data_quality`；
3. 管理员后台导出的汇总 CSV 和操作明细 CSV。

## 5. 推荐统计处理

正式论文分析时建议：

1. 主分析优先纳入 `record_validity = 有效` 的记录；
2. 若同一参与者同一阶段重复作答，主分析保留 `attempt_order = 1`；
3. 培训前后配对分析优先纳入三阶段完整完成者；
4. `failure`、`timeout` 不应直接视为无效数据，而应作为能力表现结果保留；
5. 对“需核查记录”可做敏感性分析。
