# V1.2.8 登记界面去提示与结束后自动回跳版

## 本版更新

1. 登记界面去提示化
   - 删除登记页副标题中的“药物诱发过敏反应识别与初始处置能力评价”。
   - 访问码页面改为中性表述：“系统仅用于护理教学、培训与科研，不用于临床诊疗决策。”
   - 登记后进入阶段任务页时，不再展示评分模块、肾上腺素、补液、CPR等可能提示受试者答案的版本说明。
   - 阶段任务页仅保留系统使用说明：核对信息、点击开始、独立操作、勿刷新页面。

2. 基线后补充问卷保持不变
   - “是否接受过相关培训、是否参加过模拟/虚拟仿真培训、是否处理过真实病例”仍只在基线评估操作完成并锁定评分后出现一次。
   - 登记页、模拟培训结束后、培训后考核结束后均不再显示这三项问题。

3. 每阶段完成后自动回跳登记界面
   - 基线评估：完成情境操作后先进入基线后补充问卷；补充问卷提交后自动保存并返回登记界面。
   - 模拟培训：训练完成并保存后自动返回登记界面。
   - 培训后考核：考核完成并保存后自动返回登记界面。
   - 返回登记界面后显示阶段完成提示，便于选择下一阶段或下一位受试者。

4. 评分引擎保持 V1.2.7 模块边界评分逻辑
   - 总分100分。
   - 抢救启动模块、初始支持与评估模块、首次肌注肾上腺素模块、肾上腺素后标准处理模块、复评交班沟通模块保持不变。
   - 再次肌注肾上腺素仍仅作为异常路径条件触发。

## 基础校验

- streamlit_app.py 语法检查通过。
- engine.py 语法检查通过。
- 标准路径脚本测试：普通路径可达100/100，repeat_epinephrine 不触发，结束条件为两次复评 + 家属沟通 + SBAR交接。
