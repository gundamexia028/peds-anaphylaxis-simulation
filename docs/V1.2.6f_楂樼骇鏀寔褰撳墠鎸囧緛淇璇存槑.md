# V1.2.6f 高级支持当前指征修正说明

本版本基于 V1.2.6e 继续修正，25分标准主路径不变。

## 修正背景

在标准路径已完成、患儿最终生命体征稳定且得分25/25时，系统仍可能提示“达到高级支持条件但未联系高级支持”。原因是早期短暂低氧或低血压曾触发 `advanced_support_indicated`，后续病情稳定后该标记没有被清除。

## 修正原则

1. 区分“当前仍需要高级支持”和“历史曾短暂触发高级支持条件”。
2. 最终过程性安全缺陷只依据当前仍适用状态。
3. 历史触发只作为研究日志和探索性指标保留。

## 新增/调整字段

- `advanced_support_indicated_current`：当前是否仍达到高级支持条件。
- `advanced_support_indicated_ever`：历史上是否曾达到高级支持条件。
- `advanced_support_indicated_time_sec`：首次触发高级支持条件的时间。
- `advanced_support_indicated_reason`：首次触发原因。
- `advanced_support_current_reason`：当前触发原因；稳定后为空。
- `advanced_support_contact_time_sec`：联系高级支持时间。
- `advanced_support_contact_reason`：联系时对应原因。

## 最终安全缺陷判定

只有满足以下条件时，最终报告才提示未联系高级支持：

```text
advanced_support_indicated_current = true
且
advanced_support_contacted = false
```

如果仅为历史短暂触发，但最终病情稳定，则不再提示为过程性安全缺陷。
