# 儿科药物诱发过敏反应动态分支虚拟仿真训练系统｜V1.2.5 输液场景双复评定版候选

本版本在 V1.1.4 流程锁定版基础上继续升级，用于支撑：

> 基于动态分支虚拟仿真平台的多院区儿科护士药物诱发过敏反应识别与初始处置能力现状、影响因素及培训效果研究

## 本版定位

V1.2.5 是“逻辑定版候选”，重点解决输液场景下评分路径、双复评、剂量/容量输入和特殊路径误判问题。  
本版仍然沿用 Supabase `training_records` 表，不强制新增数据库顶层字段；新增研究字段保存在 `full_report` JSON 中，并由管理员后台导出 CSV 时展开。

## 主要更新

1. **流程锁定继续保留**
   - 基线评估：考试模式 + 初始病例
   - 模拟培训：训练模式 + 初始病例
   - 培训后考核：考试模式 + Variant A
   - 受试者不能自行选择运行模式、病例脚本或随机种子。

2. **输液场景逻辑修正**
   - 删除“建立/确认静脉通路”作为标准得分项。
   - 系统默认患儿已经处于静脉输注过程中。
   - 正确操作改为“停用可疑药物并保留抢救通路”。
   - “拔除静脉通路”保留为错误路径。

3. **儿童肾上腺素剂量规则更新**
   - 1:1000 肾上腺素。
   - 0.01 mg/kg。
   - 儿童单次最大 0.3 mg。
   - 正确判定范围：目标剂量 ±0.01 mg。
   - 低于目标剂量：剂量不足，不产生有效一线治疗效果。
   - 高于目标剂量但 ≤0.3 mg：记录为剂量不准确/用药安全缺陷，不给肾上腺素剂量分。
   - >0.3 mg：严重用药安全事件。

4. **快速补液容量输入**
   - 点击“快速补液”后需要输入 ml 数。
   - 合理范围：10–20 ml/kg。
   - 单次最大 500 ml。
   - 拔除静脉通路后再补液，判定为流程矛盾，补液无效。

5. **双复评按钮**
   - 原“复评记录”拆分为：
     - 第一次复评
     - 第二次复评
   - 第一次复评：完成有效肌注肾上腺素 + 有效快速补液后才计为有效。
   - 第二次复评：第一次有效复评后、告知家属前完成才计为有效。
   - 取消 5–15 min 的硬性卡点；时间仍保存在日志中用于后续分析。

6. **特殊路径条件性记录**
   - 再次肌注肾上腺素：仅在第一次复评后仍未缓解时作为特殊路径。
   - 联系高级支持：仅在难治趋势或濒危表现时作为升级路径。
   - CPR：仅在心肺骤停状态下作为正确路径。
   - 雾化肾上腺素：仅在喉鸣/上气道受累时作为辅助路径。
   - 抗组胺药、糖皮质激素：作为辅助治疗记录，不能替代肌注肾上腺素。

7. **前台显示简化**
   - 不向受试者显示内部病情分级。
   - 不显示 `no_score_not_indicated_yet` 等内部英文状态。
   - 当前症状使用自然中文临床描述。
   - 未做雾化支扩时，不显示“无异常”，而是保留“咳嗽/轻微喘息”等残余表现。

8. **科室列表更新**
   - 保留：呼吸科、感染科、肾脏科、血液科、心血管科、神经科、PICU。
   - 新增：消化科。

## 部署说明

将本文件夹内容上传至 GitHub 仓库根目录，覆盖旧版文件：

- `streamlit_app.py`
- `requirements.txt`
- `peds_anaphylaxis_sim/`
- `.streamlit/`
- `docs/`
- `README.md`

提交后，Streamlit Cloud 会自动重新部署。原来的 Secrets 继续使用，无需重新配置。

```toml
APP_ACCESS_CODE = "peds2026"
ADMIN_PASSWORD = "admin2026"

SUPABASE_URL = "https://xxxx.supabase.co"
SUPABASE_SERVICE_ROLE_KEY = "..."
SUPABASE_TABLE = "training_records"
```
